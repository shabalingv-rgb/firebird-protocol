# Инструкция для AI-моделей (Qwen, Claude, Gemini и др.)

## 🎯 Цель

Этот документ помогает AI-моделям быстро понять контекст проекта **Firebird Protocol** и генерировать корректный код для Godot 4.6 + C# + Firebird SQL.

```
Проект: Firebird Protocol
Движок: Godot 4.6 (.NET)
Языки: C# (БД/логика) + GDScript (UI/сцены)
```

## 🔑 Критические факты

### 1. Версии SDK
- **Godot.NET.Sdk:** `4.6.0` (НЕ `4.3.0`!)
- **.NET Target:** `net8.0`
- **Firebird Client:** `10.0.0`

### 2. Именование autoload-ов
```
GameState       → GDScript (scripts/global/GameState.gd)
QuestManager    → GDScript (scripts/system/quest_manager.gd)
EmailSystem     → GDScript (scripts/system/email_system.gd)
GuideSystem     → GDScript (scripts/system/guide_system.gd)
DatabaseManager → C# (scripts/system/FirebirdDatabase.cs) ⚠️
GlobalInput     → GDScript (scripts/system/global_input.gd)
```

### 3. Регистр столбцов Firebird
Firebird возвращает столбцы в **ВЕРХНЕМ РЕГИСТРЕ**. Всегда проверяйте оба варианта:
```csharp
// C#
var dayId = row.ContainsKey("DAY_ID") ? row["DAY_ID"] : row["day_id"];
```
```gdscript
# GDScript
var day = progress.get("CURRENT_DAY", progress.get("current_day", 1))
```

### 4. Firebird ≠ SQLite
| Фича | SQLite | Firebird 5.0 |
|------|--------|--------------|
| AUTO_INCREMENT | `AUTOINCREMENT` | `GENERATED BY DEFAULT AS IDENTITY` |
| UPSERT | `ON CONFLICT ... DO UPDATE` | `EXECUTE BLOCK` или UPDATE→INSERT |
| LIMIT | `LIMIT 10` | `SELECT FIRST 10` |
| Строки | `"строка"` или `'строка'` | Только `'строка'` |
| TRUE/FALSE | `1/0` | `1/0` (SMALLINT) |
| Boolean | `BOOLEAN` | `SMALLINT` |

> ❌ **НЕ генерируй SQLite-синтаксис для этого проекта!**

## 🏗 Архитектура взаимодействия

### GDScript → C# (DatabaseManager)
```gdscript
# Получить письма для дня
var emails = DatabaseManager.GetEmailsForDay(1)

# Получить задание для письма
var quest = DatabaseManager.GetQuestForEmail(1)

# Выполнить SQL (терминал)
var result = DatabaseManager.ExecuteQuery("SELECT * FROM employees")

# Загрузить прогресс
var progress = DatabaseManager.LoadPlayerProgress(1)

# Сохранить прогресс
DatabaseManager.SavePlayerProgress(role, day, violations, flags, quests)
```

### C# → Firebird SQL
```csharp
// SELECT
var result = ExecuteQuery("SELECT * FROM game_days ORDER BY day_number");

// INSERT/UPDATE
ExecuteNonQuery("INSERT INTO game_days (role, day_number) VALUES ('employee', 6)");

// С параметрами (безопасно!)
using var cmd = new FbCommand("SELECT * FROM emails WHERE day_id = @day", _connection);
cmd.Parameters.Add("@day", FbDbType.Integer).Value = dayId;
```

### Сигналы C# → GDScript
```csharp
// В C# (FirebirdDatabase.cs)
[Signal]
public delegate void DatabaseReadyEventHandler();

EmitSignal(SignalName.DatabaseReady);
```

```gdscript
# В GDScript
DatabaseManager.database_ready.connect(_on_database_ready)
```

## 📝 Правила генерации кода

### Для C#
1. **Всегда** использовать `Godot.Collections` для взаимодействия с GDScript:
   ```csharp
   using GArray = Godot.Collections.Array;
   using GDictionary = Godot.Collections.Dictionary<string, Variant>;
   ```

2. **Типы возврата** для GDScript-совместимости:
   - Массив → `GArray` (НЕ `List<T>`)
   - Словарь → `GDictionary` (НЕ `Dictionary`)

3. **Конвертация типов** через `Variant.CreateFrom()`:
   ```csharp
   row[field] = value switch {
       int v => Variant.CreateFrom(v),
       string v => Variant.CreateFrom(v),
       _ => Variant.CreateFrom(value.ToString())
   };
   ```

4. **Обработка ошибок** — использовать `GD.PrintErr()`:
   ```csharp
   catch (Exception e) {
       GD.PrintErr("❌ Ошибка: ", e.Message);
   }
   ```

### Для GDScript
1. **Вызов C# методов** — использовать точные имена (case-sensitive):
   ```gdscript
   DatabaseManager.GetEmailsForDay(1)  # ✅
   DatabaseManager.get_emails_for_day(1)  # ❌
   ```

2. **Типизация** — использовать статическую типизацию:
   ```gdscript
   var emails: Array = DatabaseManager.GetEmailsForDay(1)
   var quest: Dictionary = DatabaseManager.GetQuestForEmail(1)
   ```

3. **Проверка регистров** — использовать `.get()` с fallback:
   ```gdscript
   var day = progress.get("CURRENT_DAY", progress.get("current_day", 1))
   ```

### Для SQL
1. **Только Firebird-синтаксис**:
   ```sql
   -- ✅ Правильно
   SELECT FIRST 10 * FROM emails ORDER BY id;

   -- ❌ Неправильно (SQLite)
   SELECT * FROM emails LIMIT 10;
   ```

2. **UPSERT через EXECUTE BLOCK**:
   ```sql
   EXECUTE BLOCK AS BEGIN
       IF (EXISTS (SELECT 1 FROM table WHERE id = 1)) THEN
           UPDATE table SET name = 'new' WHERE id = 1;
       ELSE
           INSERT INTO table (id, name) VALUES (1, 'new');
   END
   ```

3. **Строки только в одинарных кавычках**:
   ```sql
   INSERT INTO emails (subject) VALUES ('Тема');  -- ✅
   INSERT INTO emails (subject) VALUES ("Тема");  -- ❌
   ```

## 🐛 Чеклист перед генерацией

### C# код
- [ ] Класс `public partial class` и наследует `Godot.Node`?
- [ ] Методы `public` для видимости из GDScript?
- [ ] Типы возврата — `GArray`/`GDictionary` (НЕ `List`/`Dictionary`)?
- [ ] Есть обработка ошибок через `GD.PrintErr()`?
- [ ] `Variant.CreateFrom()` для конвертации типов?

### GDScript код
- [ ] Вызываются C# методы с правильным регистром?
- [ ] Используется `.get(key, fallback)` для словарей из БД?
- [ ] Типы переменных указаны явно?

### SQL код
- [ ] Синтаксис Firebird (НЕ SQLite/PostgreSQL)?
- [ ] Строки в одинарных кавычках?
- [ ] Нет `LIMIT` (использовать `SELECT FIRST`)?
- [ ] UPSERT через `EXECUTE BLOCK`?
