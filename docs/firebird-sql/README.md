# Firebird SQL 5.0 - Руководство для проекта Firebird Protocol

## Обзор

Проект использует **Firebird 5.0 Embedded** — реляционную СУБД с открытым кодом. База данных находится в файле `game_content.fdb` в корне проекта.

---

## 🔧 Установка и настройка

### Firebird Embedded 5.0
Для работы **не требуется** установка сервера Firebird. Embedded-версия работает как библиотека:

| Платформа | Файл библиотеки |
|-----------|-----------------|
| Windows | `fbclient.dll` (нужно скачать отдельно) |
| macOS | `libfbclient.dylib` (включён в проект) |
| Linux | `libfbclient.so` (нужно установить через пакетный менеджер) |

> ⚠️ **ВНИМАНИЕ:** В проекте есть `libfbclient.dylib` (macOS), но **отсутствует `fbclient.dll`** для Windows. Для разработки на Windows нужно скачать Firebird Embedded 5.0 с https://firebirdsql.org/en/firebird-5-0/

### Подключение из C#
```csharp
using FirebirdSql.Data.FirebirdClient;

string dbPath = ProjectSettings.GlobalizePath("res://game_content.fdb");
string connStr = $"Database=localhost:{dbPath};User=SYSDBA;Password=masterkey;ServerType=0;Dialect=3;";

using var connection = new FbConnection(connStr);
connection.Open();
```

**Параметры connection string:**
| Параметр | Значение | Описание |
|----------|----------|----------|
| `Database` | `localhost:path/to/file.fdb` | Путь к БД |
| `User` | `SYSDBA` | Стандартный пользователь |
| `Password` | `masterkey` | Стандартный пароль |
| `ServerType` | `0` | Embedded режим (1 = Classic, 2 = SuperServer) |
| `Dialect` | `3` | SQL диалект 3 (стандарт для Firebird 5.0) |

### Детальная схема таблиц

#### `game_days` — Игровые дни
```sql
CREATE TABLE game_days (
    id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    role VARCHAR(20) NOT NULL,          -- 'employee', 'journalist', 'manager'
    day_number INTEGER NOT NULL,        -- Номер дня (1-20)
    title VARCHAR(100),
    description VARCHAR(500),
    is_playable SMALLINT DEFAULT 1,     -- 1 = доступен, 0 = заблокирован
    alternative_activity VARCHAR(50),
    CONSTRAINT uk_game_days UNIQUE (role, day_number)
);
```

#### `emails` — Письма
```sql
CREATE TABLE emails (
    id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    day_id INTEGER,                     -- Ссылка на game_days
    sender VARCHAR(100) NOT NULL,
    sender_email VARCHAR(100),
    subject VARCHAR(200) NOT NULL,
    body VARCHAR(5000) NOT NULL,
    email_type VARCHAR(20),             -- 'quest', 'info', 'warning'
    is_required SMALLINT DEFAULT 1,
    sort_order INTEGER DEFAULT 0,
    unlock_condition VARCHAR(100)

CREATE INDEX idx_emails_day ON emails(day_id);
```

#### `quests` — Задания
```sql
CREATE TABLE quests (
    id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
```

## 🔑 Особенности Firebird 5.0

### AUTO_INCREMENT
Firebird 5.0 использует `GENERATED BY DEFAULT AS IDENTITY`:
```sql
-- ✅ Правильно (Firebird 5.0)
CREATE TABLE example (
    id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY
);

-- ❌ Старый синтаксис (не использовать)
-- CREATE GENERATOR gen_example_id;
-- SET GENERATOR gen_example_id TO 0;
```

### MERGE / UPSERT
Firebird **не поддерживает** `ON CONFLICT` как SQLite. Используйте `EXECUTE BLOCK`:
```sql
EXECUTE BLOCK AS BEGIN
    IF (EXISTS (SELECT 1 FROM sql_commands WHERE command_name = 'SELECT')) THEN
        UPDATE sql_commands SET times_used = times_used + 1 WHERE command_name = 'SELECT';
    ELSE
        INSERT INTO sql_commands (command_name, times_used) VALUES ('SELECT', 1);
END
```

Или в C#:
```csharp
// Сначала UPDATE, проверка affected rows, затем INSERT если 0
int updated = cmd.ExecuteNonQuery();
if (updated == 0) { /* выполнить INSERT */ }
```

### Регистр имён столбцов
⚠️ **ВАЖНО:** Firebird возвращает имена столбцов в **ВЕРХНЕМ РЕГИСТРЕ** по умолчанию:
```csharp
// Firebird вернёт "DAY_ID", а не "day_id"
var dayId = row["DAY_ID"];  // ✅
var dayId = row["day_id"];  // ❌ Может не работать
```

В GDScript всегда проверяйте оба варианта:
```gdscript
var day_id = progress.get("CURRENT_DAY", progress.get("current_day", 1))
```

| SQL тип Firebird | C# тип | Godot Variant |
|------------------|--------|---------------|
| INTEGER | int | int |

### Строковые литералы в SQL
```sql
-- Одинарные кавычки для строк
INSERT INTO emails (subject) VALUES ('Тема письма');

-- Экранирование одинарной кавычки
INSERT INTO emails (body) VALUES ('It''s a test');

-- Конкатенация строк
SELECT 'Привет' || ' ' || 'Мир' FROM RDB$DATABASE;

-- Специальные символы через ASCII_CHAR
SELECT ASCII_CHAR(10)  -- перевод строки (LF)
SELECT ASCII_CHAR(13)  -- возврат каретки (CR)
```

## 📝 SQL-скрипты инициализации

### Путь к скрипту
`res://scripts/database/game_content_firebird.sql`

### Структура скрипта
1. **DDL** — `CREATE TABLE` (сначала)
2. **Индексы** — `CREATE INDEX`
3. **DML** — `INSERT INTO` (данные)
4. **COMMIT** — фиксация транзакции

### Разделение команд
Команды разделяются `;`. Парсер в C# разбивает по `;`:
```csharp
sql.Split(';', StringSplitOptions.RemoveEmptyEntries)
```

> ⚠️ **Не используйте** `;` внутри строк или хранимых процедур без экранирования!

## 🐛 Распространённые ошибки

### 1. "database file appears to be in use by another process"
**Причина:** Firebird оставил lock-файл после аварийного завершения.

**Решение:** Удалить `.fdb.lock` файл в папке с БД.

### 2. "Dynamic SQL Error — token unknown"
**Причина:** Синтаксическая ошибка в SQL (часто — кавычки или `;` внутри строки).

**Решение:** Проверить SQL, экранировать одинарные кавычки `''`.

### 3. "Feature not supported on this Firebird version"
**Причина:** Используется функция, недоступная в Embedded-режиме.

**Решение:** Проверить документацию Firebird 5.0 Embedded.

### 4. "Arithmetic overflow error for type VARCHAR"
**Причина:** Данные превышают длину VARCHAR поля.

**Решение:** Увеличить длину поля или проверить данные.
