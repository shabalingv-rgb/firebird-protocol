# Анализ технических возможностей и жанра игры

## Техническая архитектура

### База данных
- **Firebird SQL** - основная СУБД для хранения игровых данных
- **Qdrant** - векторное хранилище для продвинутых поисковых возможностей
- **C# backend** - серверная логика на .NET
- **Godot 4** - игровой движок для клиентской части

### Инструменты нарратива
1. **Диалоговые окна** - встроенные в Godot UI элементы
2. **Письма** - email-клиент с системой заданий
3. **Уведомления** - системные сообщения и оповещения
4. **Системные сообщения** - логи и отладочная информация
5. **Записи в базе данных** - игровые данные как часть сюжета
6. **Терминал/консоль** - ввод SQL-запросов
7. **Справочная система** - встроенная документация
8. **Судоку** - мини-игра для разнообразия
9. **Браузер** - доступ к игровым сайтам
10. **Прогресс-система** - сохранение и загрузка состояния

### Жанр и механики
- **Обучающая игра** - обучение SQL через игру
- **Симулятор работы** - офисная рутина с элементами драмы
- **Пазл-игра** - решение задач через SQL-запросы
- **Нарративная игра** - сюжет развивается через электронную почту
- **Менеджмент** - управление временем и ресурсами

## Анализ игровых систем

### Система заданий

- Прогрессия сложности
- Последствия за ошибки

### Система прогресса
- Сохранение в Firebird
- Трекинг нарушений
- Управление днями
- Система флагов и квестов
- Множественные концовки

### Образовательная система
- Встроенная справка по SQL
- Постепенное разблокирование тем
- Практические примеры
- Обратная связь

## Вопросы для синхронизации видения

### Основные вопросы
1. **Целевая аудитория**: Кто основной игрок - новички в SQL или опытные разработчики?
2. **Тон и атмосфера**: Серьезная корпоративная симуляция или комедийная пародия на офис?
3. **Сложность**: Какой уровень сложности SQL-запросов планируется?
4. **Сюжет**: Должен ли сюжет быть линейным или ветвящимся?
5. **Концовки**: Сколько концовок планируется и как они зависят от выборов игрока?

### Технические вопросы
1. **Баланс**: Как соотнести обучение и развлечение?
2. **Темп**: Сколько времени должен занимать один игровой день?
3. **Прогрессия**: Как постепенно вводить новые концепции SQL?
4. **Обратная связь**: Как система должна реагировать на ошибки игрока?
5. **Повторяемость**: Какие механики позволят игрокам возвращаться в игру?

### Нарративные вопросы
1. **Персонажи**: Сколько ключевых персонажей и как они влияют на сюжет?
2. **Конфликт**: Какой основной конфликт движет сюжетом?
3. **Тайминг**: Как события синхронизированы с игровым временем?
4. **Моральные выборы**: Должны ли быть этические дилеммы?
5. **Юмор**: Какой уровень юмора уместен в игре?

### Дизайн-вопросы
1. **Визуальный стиль**: Ретро-офис или современный дизайн?
2. **Музыка и звук**: Какую атмосферу создавать звуковым сопровождением?
3. **Интерфейс**: Как сделать интерфейс интуитивно понятным?
4. **Мини-игры**: Сколько и каких мини-игр включить?
5. **Доступность**: Какие опции доступности нужны?
