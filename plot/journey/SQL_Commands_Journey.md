# Журнал SQL‑команд Firebird для ролей

## Обозначения
- **Начальный уровень** – команды, которые обычно используют junior‑разработчики и начинающие аналитики.  
- **Средний уровень** – запросы, требующие более глубокого понимания схемы и возможности работы с несколькими таблицами.  
- **Продвинутый уровень** – сложные DDL/DML‑операции, работа с транзакциями, хранимыми процедурами и оптимизацией.

---

## 1. Сотрудник (начальный + часть среднего)

### Начальный уровень (≈ 12 команд)
| Команда | Пример задания (одно предложение) |
|---------|-----------------------------------|
| `SELECT ... FROM ...` | Вывести список всех сотрудников, работающих в отделе «Анализ». |
| `INSERT INTO ... VALUES (...)` | Добавить нового сотрудника в таблицу `EMPLOYEES`. |
| `UPDATE ... SET ... WHERE ...` | Обновить статус текущего проекта у сотрудника с ID = 5. |
| `DELETE FROM ... WHERE ...` | Удалить из базы тестового пользователя, который больше не работает. |
| `CREATE TABLE ...` | Создать таблицу `PROJECT_LOGS` для хранения логов изменений. |
| `CREATE INDEX ... ON ...` | Создать индекс `IDX_EMP_STATUS` на поле `STATUS` таблицы `EMPLOYEES`. |
| `GRANT SELECT, INSERT ON ... TO ...` | Предоставить пользователю `analyst` права на чтение и вставку в таблицу `PROJECTS`. |
| `REVOKE ... FROM ...` | Отозвать права на изменение таблицы `FINANCE` у группы `finance_user`. |
| `COMMIT` | Зафиксировать изменения после массовой загрузки данных. |
| `ROLLBACK` | Отменить изменения, если загрузка данных завершилась ошибкой. |
| `SHOW TABLES` | Вывести список всех пользовательских таблиц в текущей базе. |
| `DESCRIBE ...` (Firebird: `SELECT * FROM RDB$RELATIONS WHERE ...`) | Посмотреть структуру таблицы `EMPLOYEES`. |

### Средний уровень (≈ 8 команд)
| Команда | Пример задания |
|---------|----------------|
| `SELECT ... FROM ... JOIN ...` | Вывести список сотрудников и их проекты, соединив `EMPLOYEES` и `PROJECTS`. |
| `SELECT ... GROUP BY ...` | Посчитать количество выполненных проектов по каждому отделу. |
| `SELECT ... HAVING ...` | Оставить только те отделы, в которых выполнено более 5 проектов. |
| `SELECT ... ORDER BY ... LIMIT ...` | Вывести топ‑5 сотрудников с наибольшим количеством выполненных задач. |
| `SELECT ... LIKE ...` | Найти всех сотрудников, чьи имена начинаются с «Ив». |
| `SELECT ... IN (...)` | Получить проекты, коды которых находятся в списке `[101,102,103]`. |
| `SELECT ... BETWEEN ... AND ...` | Выбрать проекты, даты начала которых попадают в диапазон «2023‑01‑01»‑«2023‑12‑31». |
| `SELECT ... IS NULL` | Найти сотрудников, у которых не заполнено поле `DEPARTMENT`. |

## 2. Журналист (много начального, часть среднего, немного продвинутого)

### Начальный уровень (≈ 15 команд)
| Команда | Пример задания |
|---------|----------------|
| `SELECT ... FROM ...` | Вывести последние 10 новостей из таблицы `NEWS`. |
| `INSERT INTO ... VALUES (...)` | Добавить новостную статью в таблицу `ARTICLES`. |
| `UPDATE ... SET ... WHERE ...` | Пометить статью как «опубликовано» после проверки фактов. |
| `DELETE FROM ... WHERE ...` | Удалить черновик статьи, который не прошёл редактуру. |
| `CREATE TABLE ...` | Создать таблицу `SOURCES` для хранения ссылок на источники. |
| `CREATE INDEX ... ON ...` | Создать индекс `IDX_SOURCE_NAME` на поле `NAME` таблицы `SOURCES`. |
| `GRANT SELECT ON ... TO ...` | Предоставить читателям доступ к таблице `SOURCES`. |
| `REVOKE ... FROM ...` | Отозвать доступ к конфиденциальным источникам у редакторов. |
| `COMMIT` | Сохранить изменения после массовой загрузки новостных заголовков. |
| `ROLLBACK` | Отменить загрузку, если данные новостей содержат ошибки. |
| `SHOW TABLES` | Проверить, какие таблицы доступны для работы с новостным модулем. |
| `DESCRIBE ...` | Посмотреть структуру таблицы `ARTICLES`. |
| `SELECT ... FROM ... WHERE ... ORDER BY ...` | Вывести новости, отсортированные по дате публикации (новые сверху). |
| `SELECT ... FROM ... WHERE ... LIKE ...` | Найти статьи, содержащие слово «кризис». |

### Средний уровень (≈ 10 команд)
| Команда | Пример задания |
|---------|----------------|
| `SELECT ... FROM ... JOIN ...` | Соединить `ARTICLES` и `SOURCES`, чтобы вывести имя источника к каждой статье. |
| `SELECT ... GROUP BY ...` | Посчитать количество статей, опубликованных по каждому источнику. |
| `SELECT ... HAVING ...` | Оставить только те источники, которые предоставили более 5 статей. |
| `SELECT ... ORDER BY ... LIMIT ...` | Вывести топ‑3 самых читаемых статей за неделю. |
| `SELECT ... LIKE ...` | Найти статьи, заголовки которых содержат слово «политика». |
| `SELECT ... IN (...)` | Выбрать статьи, IDs которых находятся в списке `[12,15,18,22]`. |
| `SELECT ... BETWEEN ... AND ...` | Найти статьи, даты публикации которых попадают в текущий месяц. |
| `SELECT ... IS NULL` | Найти статьи, у которых не указано имя автора. |
| `SELECT ... CAST(... AS ...)` | Привести поле `TEXT` к типу `VARCHAR(2000)` для вывода в отчёте. |
| `SELECT ... COALESCE(...)` | Вывести значение `AUTHOR_NAME`, если оно не NULL, иначе «Аноним». |

### Продвинутый уровень (≈ 5 команд)
| Команда | Пример задания |
|---------|----------------|
| `SELECT ... FROM ... INTERSECT ...` | Найти статьи, которые одновременно относятся к темам «экономика» и «технологии». |
| `SELECT ... FROM ... EXCEPT ...` | Выделить уникальные статьи, не попавшие в подборку «Топ‑10». |
| `CREATE PROCEDURE ...` | Создать хранимую процедуру `GET_TOP_ARTICLES(limit INT)` для получения топ‑N статей. |
| `EXECUTE PROCEDURE ...` | Вызвать `GET_TOP_ARTICLES(5)` и вывести результат в отчёт. |
| `CREATE TRIGGER ...` | Настроить триггер, который автоматически добавляет запись в журнал аудита при изменении статуса статьи. |

## 3. Руководитель (часть среднего + продвинутый)

### Средний уровень (≈ 8 команд)
| Команда | Пример задания |
|---------|----------------|
| `SELECT ... FROM ... JOIN ...` | Соединить таблицы `EMPLOYEES`, `PROJECTS` и `DEPARTMENTS` для получения отчёта о распределении бюджета. |
| `SELECT ... GROUP BY ... HAVING ...` | Посчитать количество проектов в каждом отделе и оставить только те, где бюджет превышает 1 млн. |
| `SELECT ... ORDER BY ... LIMIT ...` | Вывести топ‑3 отделов по сумме расходов. |
| `SELECT ... LIKE ...` | Найти проекты, названия которых содержат слово «ИННОВАЦИЯ». |
| `SELECT ... IN (...)` | Выбрать проекты, коды которых находятся в списке, переданном из внешнего отчёта. |
| `SELECT ... BETWEEN ... AND ...` | Найти проекты, даты начала которых находятся в текущем финансовом году. |
| `SELECT ... IS NULL` | Найти проекты, у которых не заполнено поле `MANAGER_ID`. |
| `SELECT ... CAST(... AS ...)` | Привести поле `BUDGET` к типу `DECIMAL(15,2)` для расчётов. |

### Продвинутый уровень (≈ 8 команд)
| Команда | Пример задания |
|---------|----------------|
| `CREATE PROCEDURE ...` | Создать процедуру `UPDATE_PROJECT_STATUS(p_id INT, new_status VARCHAR(20))` для изменения статуса проекта. |
| `EXECUTE PROCEDURE ...` | Вызвать `UPDATE_PROJECT_STATUS(42, 'COMPLETED')` и зафиксировать изменения. |
| `CREATE TRIGGER ...` | Настроить триггер, который автоматически отправляет уведомление в чат при изменении статуса проекта. |
| `CREATE INDEX ... ON ...` (выраженное) | Создать индекс на вычисляемом поле `PROJECT_YEAR = EXTRACT(YEAR FROM START_DATE)`. |
| `CREATE CONSTRAINT ...` | Добавить ограничение `CHECK (BUDGET >= 0)` к таблице `PROJECTS`. |
| `CREATE VIEW ... WITH CHECK OPTION` | Создать представление `V_PENDING_PROJECTS` только для проектов со статусом «IN_PROGRESS» и запретить изменения, нарушающие это условие. |
| `CREATE TABLE ... AS SELECT ...` | Создать таблицу `ARCHIVE_PROJECTS` как копию завершённых проектов для долгосрочного хранения. |
| `SET TRANSACTION ...` | Настроить транзакцию с уровнем изоляции `READ COMMITTED` перед запуском массового обновления бюджета. |

### Итоги распределения
- **Сотрудник**: 12 + 8 = 20 команд (начальный + средний уровень).  
- **Журналист**: 15 + 10 + 5 = 30 команд (много начального, часть среднего, небольшое продвинутое).  
- **Руководитель**: 8 + 8 = 16 команд (средний + продвинутый).  

Всего в журнале **66** SQL‑команд, распределённых по ролям в соответствии с требованиями. Каждая команда сопровождается коротким примером использования, что облегчает её внедрение в игровые задания.
