# Дневник разработки сюжета «Firebird Protocol»

## Назначение
Этот набор markdown‑файлов служит удобным «многофайловым дневником» для разработки и поддержки сюжета игры. Каждый файл отвечает за отдельный аспект повествования, что позволяет быстро находить нужную информацию как человеку‑разработчику, так и ИИ‑модели.

## Структура папки `plot/journey`
```
plot/journey/
├── README.md               # Описание структуры (вы сейчас читаете его)
├── 00_Общее_видение.md     # Краткое философское видение, тон, мрачный налёт
├── 01_Роли_и_акты.md       # Описание ролей, актов, ключевых дней
├── 02_Система_выборов.md   # Таблицы опций ответов на письма, без указания «правильного» пути
├── 03_Дневники_ролей/      # Подпапка с дневниками каждой роли (по дням)
│   ├── Сотрудник.md
│   ├── Журналист.md
│   └── Руководитель.md
├── 04_Концовки_и_флаги.md # Описание всех story‑flags и условий их разблокировки
└── 05_Случайные_события.md # Таблица случайных событий, их вероятности и влияние
```

## Как пользоваться
- **Разработчики** открывают нужный файл, вносят правки, добавляют новые варианты ответов или события.
- **Модели** могут быстро загрузить конкретный файл (например, `plot/journey/03_Дневники_ролей/Сотрудник.md`) и получить полную картину текущего дня, доступных вариантов и их последствий.
- При необходимости добавляются новые файлы в соответствующие разделы, но структура сохраняется неизменной.
