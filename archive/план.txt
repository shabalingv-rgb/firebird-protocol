## 🎮 **О ПРОЕКТЕ**

**Firebird Protocol** — образовательная игра-симулятор для обучения SQL на примере работы в НИИ "Файербёрд" (СССР, 1989 год). Игрок работает с базой данных, выполняет задания, расследует заговор.

### **Жанр:** 
- Симулятор/головоломка
- Обучение SQL через геймификацию
- Сюжетная линия с моральным выбором

---

## 🛠 **ТЕХНОЛОГИЧЕСКИЙ СТЕК**

### **Основное:**
- **Движок:** Godot Engine 4.6.1.stable.mono (C# версия)
- **Языки:** GDScript + C# (.NET 10.0)
- **СУБД:** Firebird 5.0 Embedded
- **Библиотека:** FirebirdSql.Data.FirebirdClient 10.0.0

### **Почему выбрали Firebird:**
- ✅ Встроенный режим (Embedded) — не требует сервера
- ✅ Кроссплатформенность
- ✅ SQL-стандарт (легко учить)
- ✅ Лёгкая интеграция в игру
- ✅ Историческая достоверность (СССР, 1989)

### **Альтернативы которые рассматривали:**
- SQLite плагин для Godot — **отклонили** (возвращает boolean вместо данных)
- JSON файлы — **отложили** на будущее (слишком просто для концепции)

## 💡 **РЕКОМЕНДАЦИИ ДЛЯ QWEN CODER**

### **Когда подключится БД:**
1. Протестировать `LoadPlayerProgress()` — должен вернуть прогресс
2. Проверить `GetEmailsForDay(1)` — должно вернуть 1 письмо
3. Запустить игру и проверить почту


### **Альтернативный план (если Firebird не работает):**

**JSON подход (1-2 часа):**
```csharp
// Вместо БД использовать JSON файлы
var emails = JsonConvert.DeserializeObject<List<Email>>(File.ReadAllText("emails.json"));
```

**SQLite подход (3-4 часа):**
- Удалить FirebirdSql
- Установить `sqlite-net-pcl` или `Microsoft.Data.Sqlite`
- Переписать строки подключения
- Конвертировать `.fdb` в `.db`

**Что вызвало проблемы:**
- ❌ SQLite плагин для Godot (возвращал boolean)

**Советы:**
1. Не использовать SQLite плагин `godot-sqlite` — он возвращает boolean для SELECT
2. FirebirdSql требует точного формата строки подключения
3. Godot C# требует полной пересборки при изменениях (`dotnet clean && dotnet build`)
4. Методы C# экспортируются с PascalCase, не snake_case
